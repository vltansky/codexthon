# Security policy

Do not open a public issue for a suspected vulnerability or exposed credential.

Report security issues through the repository's [private vulnerability reporting](https://github.com/vltansky/codexthon/security/advisories/new). Include affected paths, reproduction steps, and impact, but do not include real attendee data or active credentials.

If a credential was committed, revoke or rotate it immediately. Deleting it from the current branch does not remove it from Git history.
